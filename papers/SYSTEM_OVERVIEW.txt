PROJECT SUMMARY
================================================================================

A comprehensive MATLAB simulation demonstrating multi-drone detection and
tracking using an 8×8 digital array radar at 10 GHz with adaptive beamforming
and real-time beam steering dynamics visualization.

KEY SPECIFICATIONS:
  - 4 Drones with random initial positions and velocities
  - 8×8 Digital Array (64 elements) operating at 10 GHz
  - Adaptive phase-shift beamforming (per-pulse steering)
  - Micro-Doppler analysis from blade rotation
  - Range-Doppler processing for multi-target detection
  - Comprehensive beamforming analysis and visualization

RADAR SYSTEM SPECIFICATIONS
================================================================================

TRANSMITTER:
  Carrier Frequency       : 10 GHz (X-band)
  Waveform               : LFM Chirp (Linear Frequency Modulation)
  Chirp Bandwidth        : 10 MHz
  Pulse Duration         : 20 μs
  Pulse Repetition Freq  : 5 kHz
  Peak Power             : 1500 W
  Transmit Gain          : 25 dB

RECEIVER:
  Sample Rate            : 25 MHz
  Receiver Gain          : 25 dB
  Noise Figure           : 2 dB
  Receiver Preamp        : Low-noise amplifier model

ANTENNA ARRAY:
  Type                   : 8×8 URA (Uniform Rectangular Array)
  Elements               : 64 (8 rows × 8 columns)
  Element Spacing        : λ/2 = 1.5 cm (half-wavelength)
  Physical Dimensions    : 12 cm × 12 cm
  Theoretical Gain       : 18.06 dB (coherent combination)
  3dB Beamwidth          : ~3° (azimuth & elevation)
  Sidelobe Level         : ~-20 dB (uniform weighting)

WAVEFORM PARAMETERS:
  Range Resolution       : 15 m
  Max Unambiguous Range  : 30 km
  Max Unambiguous Vel    : 75 m/s
  Range Bin Width        : 15 m
  Doppler Bin Width      : 9.77 Hz

DRONE SPECIFICATIONS
================================================================================

NUMBER OF DRONES: 4 (randomly positioned)

POSITION INITIALIZATION (Random):
  X: 300 - 800 m (range from radar)
  Y: -200 - +200 m (lateral)
  Z: 80 - 150 m (altitude)

VELOCITY INITIALIZATION (Random):
  Vx: -10 - +10 m/s
  Vy: -10 - +10 m/s
  Vz: -2 - +2 m/s (altitude change)

QUADCOPTER GEOMETRY (Per Drone):
  Total Scatterers       : 9
    ├─ Body              : 1 (center of mass)
    ├─ Blade Tips        : 8 (2 blades × 4 rotors)
    
  Body RCS               : 0.2 m²
  Blade RCS              : 0.01 m² each
  Rotor Radius           : 0.12 m (blade length)
  Rotor Configuration    : 4 rotors at ±0.25m offset (diamond pattern)
  Blade Count            : 2 blades per rotor
  Blade Rate             : 150 RPM = 942 rad/s
  Blade Flash Frequency  : 10 Hz (2 blades × 150 RPM / 60)
  Blade Tip Speed        : 11.3 m/s

SIMULATION PARAMETERS
================================================================================

SCENARIO:
  Simulation Duration    : 10 seconds
  Total Pulses           : 50,000 (at 5 kHz PRF)
  Samples per Pulse      : 5,000 (at 25 MHz sample rate)

PROCESSING PIPELINE:
  1. Drone Kinematics    : Position/velocity/angle computation
  2. Transmission        : LFM chirp modulation & radiation
  3. Propagation         : Free-space channel (path loss + delay)

  6. Beamforming         : Phase-shift digital combining
  7. Matched Filtering   : Range compression
  8. Range Tracking      : Peak detection & signal extraction
  9. Analysis            : Micro-Doppler & SNR computation

MEMORY USAGE:
  RX Signal Buffer       : 2 GB (complex double, 5000 × 50000)
  MF Output              : 2 GB
  Steering Angles        : 100 MB
  Position History       : 200 MB
  TOTAL ESTIMATED        : ~5 GB

COMPUTATIONAL COST:
  Per-Pulse Operations   : ~40 K (beamforming + MF)
  Total Simulation       : ~2 Billion operations
  Typical Runtime        : 10-15 minutes (i7/Ryzen 5+)

KEY PHYSICS MODELS IMPLEMENTED
================================================================================

1. FREE-SPACE PROPAGATION
   - Two-way path loss (radar equation)
   - Propagation delay (round-trip time)
   - Phase rotation due to propagation
   - Distance-dependent signal attenuation

2. RADAR CROSS SECTION (RCS)
   - Constant RCS per scatterer
   - Composite RCS (body + 8 blade tips)
   - Simplification: frequency-independent, angle-independent
   - Real systems would use frequency-dependent models

3. DOPPLER EFFECT
   - Body radial velocity Doppler
   - Blade rotation micro-Doppler modulation
   - Sideband creation (Doppler ± multiples of blade flash frequency)
   - Unambiguous velocity range (±75 m/s at 5 kHz PRF)

4. DIGITAL ARRAY BEAMFORMING
   - Steering vector generation (phase-shift beamformer)
   - Per-element signal combination
   - Gain (18 dB coherent combination of 64 elements)
   - Sidelobe suppression (uniform weighting)

5. WAVEFORM PROCESSING
   - LFM chirp generation
   - Matched filter impulse response
   - Range compression (20:1 compression ratio)
   - Group delay correction

6. NOISE MODEL
   - Receiver thermal noise (2 dB noise figure)
   - White noise added to received signals
   - Impacts SNR but doesn't prevent detection (strong signals)

TYPICAL RESULTS INTERPRETATION
================================================================================

Range Tracking:
  ✓ All drones show smooth range tracks
  ✓ Detected range (cyan dots) overlays with ground truth
  ✓ Different slopes indicate different closing/opening rates
  ✗ If jagged: Check matched filter group delay correction

Radial Velocity:
  ✓ Values typically ±20-30 m/s
  ✓ Positive = receding, negative = approaching
  ✓ Smooth temporal variation
  ✗ If saturated: Check PRF settings for velocity ambiguity

Micro-Doppler Spectrogram:
  ✓ Horizontal smear around 0 Hz (body motion)
  ✓ Modulation lines spaced ~10 Hz apart (blade flash)
  ✓ Higher intensity = stronger signal
  ✗ If blank: Check tracked signal extraction

SNR Evolution:
  ✓ Higher when drone is closer (1/range dependence)
  ✓ Varies ±5 dB due to blade orientation effects
  ✓ Generally decreasing trend (drones moving away)
  ✗ If extremely negative: Check receiver gain settings

3D Paths:
  ✓ 4 distinct paths starting from different positions
  ✓ Mix of approaching & receding trajectories
  ✓ Altitude variations visible
  ✗ If all overlapping: Check random seed / initialization

Beamforming Angles:
  ✓ Smooth time variation (adaptive steering)
  ✓ Azimuth: typically ±45° to ±60°
  ✓ Elevation: typically 10°-30°
  ✗ If constant: Check angle computation in main loop

Radiation Patterns:
  ✓ Main lobe ~3° wide
  ✓ Sidelobes 20-30 dB below main lobe
  ✓ Pattern center follows steer angle
  ✗ If asymmetric: Check URA geometry

ADVANCED ANALYSIS CAPABILITIES
================================================================================

1. MICRO-DOPPLER CLASSIFICATION
   - Extract micro-Doppler signature
   - Measure blade flash frequency = 10 Hz (confirms quadcopter)
   - Different drones/vehicles have different signatures

2. ANGLE OF ARRIVAL ESTIMATION
   - Monopulse processing (currently uses true angle)
   - MUSIC or ESPRIT algorithms for angle estimation
   - Multi-target angle resolution

3. VELOCITY UNFOLDING
   - Handle ambiguous velocities (if PRF too low)
   - Use multiple PRF measurements
   - Unambiguous velocity extension technique

4. ADAPTIVE INTERFERENCE REJECTION
   - Null steering toward interference
   - MVDR (Capon) beamforming
   - Sidelobe control optimization

5. CLUTTER SUPPRESSION
   - Moving Target Indicator (MTI) filtering
   - Doppler-based clutter removal
   - Range-dependent clutter mitigation

6. SIMULTANEOUS MULTI-BEAM TRACKING
   - Form 2-4 beams simultaneously
   - Parallel processing for each beam
   - Trade-off between number of beams and SNR

TROUBLESHOOTING GUIDE
================================================================================

Issue: "Out of Memory" Error
  Cause: Large array or long simulation
  Fix: Reduce totalDuration, fs, or ura.Size

Issue: Simulation Takes > 30 Minutes
  Cause: Computer is slow or parameters too large
  Fix: Use Quick Test configuration above

Issue: All Plots Are Blank
  Cause: Signal processing error or parameter mismatch
  Fix: Check console output for warnings; run with Ndrones=1 first

Issue: Beamforming Analysis Won't Run
  Cause: multiDroneMotion.m not found
  Fix: Ensure file is in same folder and path is correct

Issue: Range Tracking Shows Constant Range
  Cause: Drones aren't moving or initialization failed
  Fix: Verify velocity values in drone initialization

See QUICKSTART.md for more troubleshooting tips and MATLAB commands.
