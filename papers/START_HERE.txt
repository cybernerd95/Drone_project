WHAT HAPPENS WHEN YOU RUN IT
================================================================================

INITIALIZATION (takes a few seconds):
  • Creates 4 drones at random positions
  • Sets up 8×8 array (64 elements)
  • Initializes LFM waveform
  • Allocates memory (~5 GB)

SIMULATION LOOP (10-15 minutes):
  For each of 50,000 pulses:
    1. Compute drone positions & look angles
    2. Transmit LFM chirp pulse
    3. Propagate through free-space channel
    4. Receive on 64 array elements
    5. Apply beamformer (digital combining)
    6. Estimate signal properties
    → Shows progress every 1000 pulses

POST-PROCESSING (takes a minute):
  • Range compression (matched filtering)
  • Range tracking
  • Spectrogram computation

PLOTTING (generates 7 figures):
  • Range tracking overlay
  • Radial velocity profiles
  • Micro-Doppler spectrogram
  • SNR evolution
  • 3D flight paths
  • Beamforming angles
  • Radiation patterns

CONSOLE OUTPUT: Performance summary
  • Detection ranges, speeds, SNR values

UNDERSTANDING THE PLOTS
================================================================================

FIGURE 1: Range Tracking
  Shows: How close each drone is (radar range)
  4 colored lines = 4 drones
  Cyan dots = matched filter detections
  What to look for: Smooth, non-overlapping tracks

FIGURE 2: Radial Velocity
  Shows: Closing/opening rates for each drone
  Positive = moving away, Negative = approaching
  What to look for: Varied velocity profiles

FIGURE 3: Micro-Doppler
  Shows: Time-frequency content (rotating blades)
  Horizontal lines = blade rotation modulation
  Spacing ~10 Hz = blade flash frequency
  What to look for: Distinctive modulation pattern

FIGURE 4: SNR Evolution
  Shows: Signal strength over time
  Higher = drone is closer or better orientation
  What to look for: Decreasing trend (drones moving away)

FIGURE 5: 3D Flight Paths
  Shows: Where drones go in 3D space
  Circles = start positions
  Squares = end positions
  What to look for: Diverse paths around radar

FIGURE 6: Beamforming Angles
  Top: Azimuth steering (horizontal angle)
  Bottom: Elevation steering (vertical angle)
  What to look for: Smooth changes (adaptive steering working)

FIGURE 7: Radiation Patterns
  4 snapshots of antenna gain pattern
  Narrow main lobe (3° wide)
  Suppressed sidelobes (~20 dB below)
  What to look for: Pattern center following steer angle

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "Undefined function or variable 'multiDroneMotion'"
SOLUTION: Ensure multiDroneMotion.m is in same folder as main script

PROBLEM: "Out of Memory" error during simulation
SOLUTION: 
  - Reduce totalDuration to 2 seconds
  - Reduce array size from [8 8] to [4 4]
  - Reduce sample rate fs from 25e6 to 12.5e6

PROBLEM: Simulation runs very slowly (> 1 hour)
SOLUTION:
  - Use faster computer
  - Reduce totalDuration
  - Reduce array size

PROBLEM: All plots are blank or show zeros
SOLUTION:
  - Check console for error messages
  - Verify PRF setting (should be 2e3 to 10e3)
  - Try with Ndrones = 1 (simpler test case)

PROBLEM: "Phased Array System Toolbox" not found
SOLUTION:
  - Open Add-Ons in MATLAB
  - Search for "Phased Array"
  - Install if available
  - Contact your IT department if unavailable

For more troubleshooting: See QUICKSTART.md "Troubleshooting Guide"
